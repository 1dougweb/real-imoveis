Visão Geral

Este documento descreve a arquitetura e os componentes do frontend desenvolvido para o sistema Laranja Real Imóveis. O frontend foi construído utilizando React com TypeScript, seguindo as melhores práticas de desenvolvimento moderno.

Ambiente de Desenvolvimento:
  • Node.js 16+
  • React 18
  • TypeScript 5
  • Vite como bundler
  • TailwindCSS para estilização
  • Shadcn/UI para componentes base

Arquitetura do Frontend

A aplicação frontend segue uma arquitetura baseada em componentes, com separação de responsabilidades e organização modular do código.

Principais Características

  • Componentização: Baseada em princípios de design atômico, organizando componentes por complexidade e responsabilidade
  • Hooks personalizados: Encapsulamento de lógica reutilizável em hooks customizados
  • React Router: Gerenciamento de rotas e navegação entre páginas
  • Estados locais e globais: Combinação de useState e useContext para gerenciamento de estado
  • Autenticação JWT: Gerenciamento de sessão do usuário baseado em tokens
  • Consumo de API REST: Integração com o backend através de serviços estruturados
  • Responsividade: Design adaptável para diferentes tamanhos de tela
  • Acessibilidade: Conformidade com diretrizes WCAG para tornar a aplicação acessível a todos os usuários

Autenticação e Autorização

O sistema de autenticação do frontend é baseado em tokens JWT, com gerenciamento de sessão e controle de acesso:

Processo de Autenticação

  1. Usuário realiza login fornecendo email e senha
  2. Backend valida as credenciais e retorna um token JWT
  3. Frontend armazena o token no localStorage
  4. Token é incluído como Authorization header em todas as requisições subsequentes
  5. Context API gerencia o estado de autenticação globalmente

Controle de Acesso

O acesso às rotas e funcionalidades é controlado com base nas funções do usuário:

// Exemplo de componente de rota protegida
function ProtectedRoute({ children, roles }) {
  const { user, isAuthenticated } = useAuth();
  const hasPermission = roles ? roles.some(role => user?.roles.includes(role)) : true;

  if (!isAuthenticated) {
    return <Navigate to="/login" />;
  }

  if (!hasPermission) {
    return <Navigate to="/unauthorized" />;
  }

  return children;
}

Integração com o Backend

A comunicação com a API do backend é centralizada em serviços especializados:

Cliente de API

Um cliente Axios configurado com interceptors para gerenciar tokens, erros e renovação de autenticação:

// src/lib/api.ts
import axios from 'axios';

const api = axios.create({
  baseURL: import.meta.env.VITE_API_URL || 'http://localhost:8000/api',
  headers: {
    'Content-Type': 'application/json',
    'Accept': 'application/json'
  }
});

// Interceptor para incluir o token JWT em todas as requisições
api.interceptors.request.use(
  (config) => {
    const token = localStorage.getItem('token');
    if (token) {
      config.headers.Authorization = `Bearer ${token}`;
    }
    return config;
  },
  (error) => Promise.reject(error)
);

export default api;

Serviços Especializados

A comunicação com a API é organizada em serviços especializados por domínio:

  • authService: Métodos para autenticação, registro e gerenciamento de sessão
  • propertyService: Operações relacionadas a imóveis
  • clientService: Operações relacionadas a clientes
  • visitService: Operações relacionadas a visitas
  • contractService: Operações relacionadas a contratos

Estado Local

Combinação de componentes personalizados e biblioteca Shadcn/UI com TailwindCSS:

// src/components/ui/Button.tsx
import React from 'react';
import { cva, type VariantProps } from 'class-variance-authority';
import { cn } from '../../lib/utils';

const buttonVariants = cva(
  'inline-flex items-center justify-center rounded-md text-sm font-medium transition-colors focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-offset-2 disabled:opacity-50 disabled:pointer-events-none',
  {
    variants: {
      variant: {
        default: 'bg-primary text-white hover:bg-primary/90',
        outline: 'border border-input bg-background hover:bg-accent hover:text-accent-foreground',
        secondary: 'bg-secondary text-white hover:bg-secondary/90',
        ghost: 'hover:bg-accent hover:text-accent-foreground',
        link: 'underline-offset-4 hover:underline text-primary',
      },
      size: {
        default: 'h-10 py-2 px-4',
        sm: 'h-9 px-3',
        lg: 'h-11 px-8',
        icon: 'h-10 w-10',
      },
    },
    defaultVariants: {
      variant: 'default',
      size: 'default',
    },
  }
);

export interface ButtonProps
  extends React.ButtonHTMLAttributes<HTMLButtonElement>,
    VariantProps<typeof buttonVariants> {
  asChild?: boolean;
}

const Button = React.forwardRef<HTMLButtonElement, ButtonProps>(
  ({ className, variant, size, asChild = false, ...props }, ref) => {
    const Comp = asChild ? 'div' : 'button';
    return (
      <Comp
        className={cn(buttonVariants({ variant, size, className }))}
        ref={ref}
        {...props}
      />
    );
  }
);
Button.displayName = 'Button';

export { Button, buttonVariants };

Deployment e Build

O processo de build e deployment foi otimizado para fornecer a melhor experiência tanto para desenvolvedores quanto para usuários finais.

Scripts Disponíveis

  • npm run dev: Inicia o servidor de desenvolvimento
  • npm run build: Cria uma versão otimizada para produção
  • npm run preview: Executa localmente a versão de produção
  • npm run lint: Verifica problemas no código
  • npm run test: Executa testes automatizados

Otimizações

  • Code Splitting: Carregamento sob demanda de partes da aplicação
  • Prefetching: Pré-carregamento de recursos para navegação mais rápida
  • Lazy Loading: Carregamento preguiçoso de imagens e componentes pesados
  • Bundle Analyzer: Análise e otimização do tamanho dos pacotes
